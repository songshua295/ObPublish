    # IO流

    Created: August 1, 2022 8:50 AM 类别: IO

    包：java.io

    # 书写注意点

    1. 创建流对象的时候异常：
      1. 写入对象会有IOException需要捕获，因为你输出输出数据了
      2. 读取对象会有FileNotFoundException 需要捕获，因为你读取文件路径要找文件
      3. 流关闭的时候会有IOException，同理，你要找文件把它关闭了
    2. 先创建的流对象要赋值null，放在try外面，try里面在对文件路径进行赋值，避免在finally关闭的时候找不到变量
    3. output操作一定要记得flush
    4. Stream结尾的是字节流，什么文件都可以操作；但是writer和reader结尾的是字符流，专用对普通文本操作，可以 直接进行String的写出

    # 流的分类

    1. 流的方向：输入流、输出流
    2. 读取数据的方式：字节流、字符流

    # IO四大家族

    1. java.io.InputStream 字符输入流
    2. java.io.OutputStream 字符输出流
    3. java.io.reader 字符输入流
    4. java.io.writer 字符输出流

    # 注意

    1. 类名以Stream结尾的是字节流，以reader/writer结尾的是字符流
    2. 所有的流都有close()关闭，用完之后要关闭，避免浪费资源
    3. 所有的输出流都有flush()方法，所有的输出流用完之后要记得刷新。刷新表示将管道中的剩余未输出数据强行输出完，作用就是清空管道 写操作必须要flush
      1. 缓冲区太大，输出流，输出流就可能偷懒，没有写入，使用flush就会强制把没有填满缓冲区的部分全部输出

    import java.io.FileInputStream;

import java.io.FileOutputStream;
import java.io.IOException;

public class CopyTest01 {
    public static void main(String[] args) {
    //     输入流对象
        FileInputStream fis=null;
    //    输出流对象
        FileOutputStream fos=null;
        try {
            fis=new FileInputStream("C:\\Users\\22305\\Downloads\\tiddlers.html");//要复制的文件
            fos=new FileOutputStream("C:\\Users\\22305\\Desktop\\复制的文件.html");//复制的文件路径

            while ((readCount=fis.read(bytes))!=-1){
                fos.write(bytes,0,readCount);//写入读取到的字节数
            }
            fos.flush();//写入文件必备的一部，刷新缓冲区，清空管道
        } catch (FileNotFoundException e) {
            e.printStackTrace();
        } catch (IOException e) {
            e.printStackTrace();
        }finally {
            if (fis!=null){
                try {
                    fis.close();
                } catch (IOException e) {
                    e.printStackTrace();
                }
            }
            if (fos!=null){
                try {
                    fos.close();
                } catch (IOException e) {
                    e.printStackTrace();
                }
            }
        }
    }
}

    import java.io.FileNotFoundException;
import java.io.FileReader;
import java.io.FileWriter;
import java.io.IOException;

public class CopyTest02 {
    public static void main(String[] args) {
        FileReader in=null;
        FileWriter out=null;
        try {
            in=new FileReader("ioTest/src/普通文本写入.txt");//被复制的普通文本文件
            out=new FileWriter("ioTest/src/复制的文本文件.txt");
            char[] chars=new char[1024*512];//1MB
            int readCount=0;//读取的字符数
            //读取的字符数不小于0，就写入
            while ((readCount=in.read(chars))!=-1){
                out.write(chars,0,readCount);//读几个字符，就写几个字符
            }
            //写完之后，避免遗漏，需要flush
            out.flush();
        } catch (FileNotFoundException e) {
            e.printStackTrace();
        } catch (IOException e) {
            e.printStackTrace();
        }finally {
            if (in!=null){
                try {
                    in.close();
                } catch (IOException e) {
                    e.printStackTrace();
                }
            }
            if (out!=null){
                try {
                    out.close();
                } catch (IOException e) {
                    e.printStackTrace();
                }
            }
        }
    }
}

    # 书写注意点

    1. readLine()可以一次读一行，末尾的换行符不算在内，所以建议采用println
    2. 只能传字符流，不能传字节流 末尾只能是Reader或者Writer，所以可以转换流，再传入
    3. 传入的FileReader参数不需要close，会自动close，只需要close()BufferedReader对象就好了

     * 复制 一个目录下的所有文件和目录到另一个目录下
     * @param src 源目录
     * @param dest 目标目录
     * @throws IOException
     */
    private static void copyAll(File src, File dest) throws IOException {
        if (src.isDirectory()) {
            //目标目录是否存在，不存在创建
            if (!dest.exists()) {
                dest.mkdirs();
            }
            String[] files = src.list();//源文件的目录
            for (String file : files) {
                //    创建目标File对象
                File srcDir = new File(src, file);
                System.out.println("源文件：\t"+srcDir);
                File destDir = new File(dest, file);
                System.out.println("后文件：\t"+destDir);
                //    递归
                copyAll(srcDir, destDir);
            //

            }

            // 文件复制
        } else {
            FileInputStream in=new FileInputStream(src);
            FileOutputStream out=new FileOutputStream(dest);
            System.out.println("srcfile:\t"+src);
            System.out.println("destfil:\t"+dest);
            System.out.println();
            //字节数组
            byte[] bytes=new byte[1024];
            int readCount=0;
            while ((readCount=in.read(bytes))!=-1){
                out.write(bytes,0,readCount);
            }
            out.flush();//清空管道
            in.close();
            out.close();
        }
    }

}

    # 序列化

    # 书写注意点

    1. 参与序列化或反序列化的对象类必须要实现Serializable，否则会出现NotSerialzableException异常

    2. 建议将序列化版本号写出来

      1

      static final long serialVersionUID = -6430539691155161871L;

    3. Serializable是一个标志性接口，JVM会对实现了这个接口的特殊处理，会自动生成一个序列化版本号，标识这个类

    4. JVM判断一个类，是先对比类名，再对比序列化版本号

    5. 不想序列化的属性可以使用transient修饰 ，HashMap中就有此种属性

    # 图解

    # 简述

    1. 类似于网络分片
    2. Serialize序列化：内存中的java对象分片放到硬盘
    3. DeSerialize 反序列化：分片的java对象从硬盘恢复到内存中
