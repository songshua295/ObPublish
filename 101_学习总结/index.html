    #Java基础

    # 前序

    通过大致17天的学习，学习了javase的基础，让我对java的一些基本特点或者使用有了轮廓。 接下来将要学习java进阶，所以就目前的学习做一下总结。

    # 一、理论

    在理论方面学习了

    1. java开发工具包JDK，包括了JREJava运行环境（包括了JVM虚拟机）
    2. 学习了java程序的执行过程：源代码执行，生成字节码，JVM解释为二进制文件，交给操作系统，操作系统与硬件进行交互
    3. 基本的环境变量设置：sysdm.cpl
    4. 三种注释方法
    5. public class和class的区别：一个文件最多一个public class，但可以有无数个class，public的class必须和文件名一致

    # 二、铺垫

    1. 变量名定义：驼峰命名；基本的关键字50个
    2. javadoc的基本使用：需要配合文档注释
    3. 不同数据类型所占用的字节大小：1248 4812 （byte short int long；float double boolean char）
    4. Scanner的使用，基本的导包方式（和那个python好像一样）
    5. 方法的调用：在一个类中，可以直接写想要调用的方法，如果不在一个类中，则采用：类名.方法
    6. break和return的区别：return可以后面不跟返回值，用来停止一个方法
    7. JVM的三个主要内存：堆 栈 方法区的基本分配，一个程序运行时在内存JVM中的情况
    8. 方法重载overload：在一个类中，书写的某个方法可能有多重变化，可以定义多个参数列表不同，方法名称相同的方法，就叫方法重载。（参数列表指的形参列表），在使用的时候，java会根据你的形参列表去判断执行哪一个具体的方法

    # 三、基本

    1. 递归：能转成循环就转成循环，耗内存。写递归嘴重要的是知道停止条件。通过调用自身去解决问题
    2. 面向对象和面向过程的特点和区别：java面向对象，耦合度低，扩展性强
    3. 对象和类的关系：通过类去实例化对象。如何去定义类，如何去实例化对象new
    4. 实例化对象，对对象进行赋值和修改
    5. 什么时候会出现空指针异常java.lang.NullPointerException，将实例化对象的那个引用置空，那么那个对象就引用任何东西了，对象所存储的东西就处于游离状态。访问引用对象的属性时就会出现空指针异常
    6. 方法调用时的参数传递，实参和形参以及局部变量的释放

    # 四、构造方法

    1. 构造方法语法格式：public 类名{}
    2. 构造方法的注意事项：手动提供有参构造之后，无参构造就不再提供，所以必须要记得把无参构造写上，除非你不用无参构造
    3. 无参数和有参数的构造方法及其使用:蔡徐坤打篮球

    # 五、封装

    1. 封装的作用：保证内部安全，屏蔽复杂，暴露简单。
    2. 基本语法：就是get和set对内部数据进行获取和修改
    3. get和set应该也是实例方法，因为需要依赖实例对象
    4. 通俗的解释：封装就是，把数据转换成一个工具。你可以通过这个工具进行修改，工具使用起来很简单，客户方便使用；而我可以通过这个工具，把里面有的不想给你看到的，不想让你有修改权利的东西给藏住不让你用，不让你看。

    # 六、this 和 static关键字

    1. 静态代码块：类加载时机
    2. 实例代码块：对象创建时机
    3. static声明之后存在方法区，类加载时就加载static定义的东西
    4. 静态方法和实例方法：实例方法依赖于实例对象，用对象.的方式调用；而静态方法加载在方法区，用类.的方法调用
    5. 即使将对象置空，引用静态变量，依旧不会出现空指针异常，因为静态变量不依赖于对象的建立
    6. 不同代码块的执行顺序：最先加载类
    7. this的内存结构:放在堆内存中
    8. this();
    9. 各种变量在内存中的分布：局部变量（栈），静态变量（方法区），实例变量（堆）

    # 七、继承

    1. 除了构造方法，其余的子类都继承，包括实例方法和get set方法都可以继承
    2. 主要作用：有了继承关系，才有后续的方法覆盖和多态
    3. java中只支持单继承
    4. 在进行重新构造子类构造器的时候，可以通过super关键字快速完成
    5. 凡是可以用“is a”来描述的，都可以考虑采用继承

    # 八、方法覆盖和多态

    1. 当继承的方法无法满足当前子类的业务需求，子类有权利对这个方法进行重新编写，进行方法覆盖override
    2. 私有方法、构造方法、静态方法无法覆盖，方法覆盖针对实例方法
    3. java程序在 编译阶段 和执行阶段 所真正参与的是不同的对象，所以叫多态。
    4. Java中允许向上向下转型
    5. 当你访问的是一个子类对象中所“特有”的方法，此时必须向下转型，使用强制转型方法
    6. 强制类型转型的风险是java.lang.ClassCastException （类型转换错误），原因是强制类型转换的两个类没有父子类型继承关系，可以通过if和instanceof关键字解决
    7. 学习采用抽象编程，而不是具体编程：主人养宠物

    # 九、super关键字

    1. this与super都是与对象有关，常常在构造方法或者实例方法中使用
    2. super后面必须跟.，因为他不是引用，不保存内存地址，而this可以直接使用
    3. 当一个构造方法没有this和super的时候，第一行默认是有super();的，表示通过子类的构造方法调用父类的无参构造方法
    4. 必须保证父类的无参构造方法是存在的
    5. 构造方法的执行顺序：超父类Object类是一定会执行的，因为任何一个类extends类，都默认继承Object类
    6. 使用super() 可以调用父类的构造方法，使用super.xxx可以调用父类的属性，使用super.方法名称（实参），可以访问父类的方法

    # 十、遗留问题

    1. 静态方法不存在方法覆盖，没有意义，因为与对象无关
    2. 私有方法无法覆盖
    3. 方法覆盖的返回值类型在基本数据类型必须一样，而在引用类型中（就是你class的那些类的对象），可以是你class的子类树上的类
