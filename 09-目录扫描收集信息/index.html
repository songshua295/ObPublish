    # st.10 目录扫描收集信息

    目录扫描可以扫描到网站的一些文件目录结构，通过目录中的文件来获取一些信息。 部署的网站的一些敏感信息 1. 配置文件：.cfg

    1. 数据文件：.sql,.tar.gz
    2. 目录：/backup,/conf,/admin 可能会泄露的信息：
    3. 数据库用户名和密码
    4. 服务器的用户名和密码
    5. 网站源码
    6. 数据库的文件
    7. 一些特殊有用的配置文件

    泄露的原因：配置不当

    1. 本地文件包含：
    2. local file inclusion（LFI）
    3. PHP 1. header.php 2. common.php 3. footer.php 4. function.php
    4. include(“路径/文件”):include("../../..")
    5. require();
    6. dvwa靶场 1. http://localhost/dvwa/vulnerabilities/fi/? page=../../phpinfo.php 2. http://localhost/dvwa/vulnerabilities/fi/?page=../../../../../../../in.txt

    # 常见的敏感目录和文件

    1. robots.txt:禁止搜索引擎爬虫爬的文件
    2. sitemap.xml ：指导搜索引擎爬哪些内容或者目录的一个文件
    3. 网站的备份文件/数据
    4. 在线压缩
      1. 路径
      2. 文件名
      3. wwwroot –20210212.zio
    5. 帝国备份王：比如1.sql，1.zip，不仅能够备份文件，还能够备份数据
    6. 后台登录的目录
    7. /admin
    8. /manage
    9. 安装包（源码）：非开源，商用的，里面可能有默认的一些9配置，比如密码之类的，可以利用
    10. 上传的目录
    11. 文件上传漏洞,如web shell
    12. /upload
    13. /upload.php
    14. mysql的管理界面
    15. MySQL
    16. web页面去管理
    17. phpadmin-密码爆破-webshell：主要是一些虚拟主机，不会给一个数据库，而是给一个数据库的web管理页面，同时没有一些登录验证，可以密码爆破等等方式进行破解
    18. 程序的安装路径 ：/install
    19. php的探针
    20. phpinfo
    21. 雅黑探针
    22. 文本编辑器：这里说的文本编辑器，指的是富文本编辑器有一些漏洞存在，比如文件上传，命令注入
    23. Ueditor：https://github.com/fex-team/ueditor
    24. kindeditor
    25. CKeditor
    26. Linux
    27. /etc/passwd ：用于存放用户以及用户组等等相关信息
    28. /etc/passwd ：存放了用户密码，使用SHA512加密，一些常规的密码可以在网上找到逆向解密工具
    29. /etc/sudoers ：sudo，存放了可以使用sudo命令的用户
    30. MacOS
    31. .DS_Store：存放了一些配置文件，可以获取到一些敏感文件的文件名、目录结构等等
    32. 编辑器的临时文件.swp：linux中就有，可能会有之前临时中断编辑的文件，可能会存在一些敏感内容
    33. 目录穿越/目录遍历
    34. Windows IIS
    35. Apache
    36. pikachu靶场
      1. http://localhost/pikachu/vul/dir/dir_list.php?title=jarheads.php
      2. http://localhost/pikachu/vul/dir/dir_list.php?title=../../../../Windows/win.ini
    37. tomcat WEB-INF：这个一般javaweb或者java项目会有
    38. WEB-INF/web.xml : Web应用程序配置文件, 描述了servlet和其他的应用组件配置及命名规则.
    39. WEB-INF/database.properties : 数据库配置文件
    40. WEB-INF/classes/ : 一般用来存放Java类文件(.class)
    41. WEB-INF/lib/ : 用来存放打包好的库(.jar)
    42. WEB-INF/src/ : 用来放源代码(.asp和.php等)
    43. 其他非常规文件：主要值得是一些用户可能会了方便自己，存的一些敏感文件
    44. secret.txt
    45. password.txt

    1. 做法：直接在域名后面拼接路径/文件名，如果返回200，就是存在
2. 扫描方法：
1. 递归：dir xxx
2. 字典：dict
3. 暴力破解：字符串挨着试
4. 爬虫
    1. robots.txt
    2. sitemap.xml
    3. 网页中的其他链接
5. fuzz（模糊测试）
    1. 字典
        1. /word
        2. /index.php?word=
3. 文件扫描的字典
1. kali：wordlists命令可以查看字典
2. dirb ：/usr/share/wordlists/dirb/common.txt
4. 工具
1. dirb
2. dirbuster
3. 御剑
4. Burp Suite
    1. Intruder--payload
    2. www.baidu.com/$bbs$
5. DirBrute
6. Dirsearch
7. Dirmap
8. wfuzz
5. 注意事项 :WAF、IDS,可能会防止你扫描，可以使用 **代理、网络空间搜索引擎**解决
6. 防御
1. 设置权限
2. 删除敏感文件
3. WAF、IDS来检测和过滤等等
