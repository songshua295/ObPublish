# 2319_TreeSet关于排序：实现comparable接口或使用比较器Compartor

💡 Comparable和Comparator怎么选择呢？ 当比较规则不会发生改变的时候，或者说**当比较规则只有1个的时候，建议实现Comparable接口。**因为这个要去实现，没下面那个方便，下面那个方便，但是每次都要作为参数传进去 如果比较规则有多个，并且需要多个比较规则之间频繁切换，建议使用Comparator接口。

# 第一种：实现comparable接口

# 关于TreeSet自定义类的排序，书写代码时需要注意

  1. 继承Comparable接口
  2. override compareTo``方法
  3. 有必要的话，把toString方法也override

# compareTo的比较规则怎么写

  1. 通过对象的属性进行比较，比如是String，可以用它本身就实现了的compareTo去进行比较，返回的一个数值能够代表他的数值
  2. 通过数值大小比较
  3. 使用if结合起来比较
  4. 无论怎么写，返回的都是一个数值，通过这个数值进行比较，一般是升序

# 原因

  1. TreeSet中存入的数据会默认的排序，因为String和Integer都是默认实现了Comparable接口，对排序的规则已经进行了一定，所以不用再写了。
  2. 但是，别的自己定义的类，没有定义规则，所以需要实现Comparable接口，不然无法排序，添加节点元素就会报错Customer cannot be cast to java.lang.Comparable

import java.util.TreeSet;

public class Test02 {
    public static void main(String[] args) {

//创建TreeSet
TreeSet<Customer> set=new TreeSet<>();
//添加节点元素
		set.add(new Customer("zhangsan",17));
        set.add(new Customer("zhangsan1",17));
        set.add(new Customer("wangmazi",16));
        set.add(new Customer("haiya",15));

//遍历
for (Customer c:set
             ) {
            System.out.println(c);
        }
    }
}
class Customer implements Comparable<Customer>{
    private String name;
    private int age;

    public Customer() {
    }

    public Customer(String name, int age) {
        this.name = name;
        this.age = age;
    }

/**
     *
     * @paramothe object to be compared.
     * @return返回数字表示这个set的排序方式：先根据年龄，再根据姓名排序
*/
@Override
    public int compareTo(Customer o) {
        if (this.age==o.age){
            return this.name.compareTo(o.name);

        }else{
            return this.age-o.age;//返回一个数值，可以表示顺序的排列，如果大于0，就排在右子树，小于0，就在左子树

}
    }

//重写toString方法，方便输出对象的值，这样就不是hashCode值了
@Override
    public String toString() {
        return "name:"+name+"\\tage:"+age;
    }
}
