禁用笔记本自带键盘四种方法

最近编辑于 Unknown

    笔记本外接键盘之后，老是误触笔记本自带键盘，整理一些禁用笔记本自带键盘的方法

    方便我的：不要使用以下命令，这是方便我的笔记

    1
2

      pnputil /removedevice "HIDVID_320F&PID_5055&MI_01&COL01\7&2D4F6D52&0&0000"
  pnputil /removedevice "HIDVID_048D&PID_C100&COL03\6&8BB26F5&0&0002"

    目前主要有以下几种办法：

    # 第一种：CMD 命令

    我的电脑这种方法不管用

    1. 使用 win + R 打开运行窗口，输入 CMD ，然后按住 Ctrl+Shift+Enter 使用管理员打开命令提示符窗口（需要注意是用管理员的身份打开）
    2. 选择如下命令进行禁用或者启用自带键盘

    shell禁用 sc config i8042prt start= disabled 启用 sc config i8042prt start= auto 最后 重启笔记本

    # 第二种：设备管理器禁用

    此方法用了很久, 因为我无法禁用, 卸载的话, 每次电脑开机都要设置一遍, 非常麻烦。

    1. 直接到设备管理进行禁用，打开设备管理的方式有很多种，比如： 命令行输入 Devmgmt.msc win+X 快捷键，然后选择 直接 win 键，然后输入设备管理器进行搜索 或者到控制面板找，这种老土的办法我一般不用
    2. 找到对应的键盘设备
    3. 这里面的任何一个键盘可能是你的笔记本键盘，所以挨着试吧，禁用或者卸载都可以。区别是卸载在重启之后会自动安装
    4. 如果卸载错了，可以通过重新扫描设备恢复已经卸载的设备

    # 第三种：使用 DevCon 工具包进行卸载

    DevCon 是 windows 官方的一个工具包，它的作用就是设备管理

    由于安装需要安装 sdk，然后找对应的文件，所以我没有具体使用，相关教程以及官方文档如下

    1. 使用 DevCon 工具安装驱动程序包
    2. DevCon Remove
    3. 用于卸载设备的批处理文件
    4. DevCon 命令行实用工具用法(安装卸载驱动)

    # 第四种：PnPUtil 命令行工具（推荐）

    # 介绍

    官网介绍： https://learn.microsoft.com/zhcn/windowshardware/drivers/devtest/pnputil

    优点：因为是自带的，不需要安装

    PnPUtil (PnPUtil.exe) 是一种命令行工具，可让管理员对 驱动程序包执行操作。 说白了，就是主要作用就是设备管理

    使用这个工具，你需要知道两个东西，不过如果是卸载设备，可以直接看我的实例

    1. 命令怎么使用
    2. 设备实例 ID 怎么找

    # 实例

    1. 打开设备管理器，右键选择你需要卸载的设备，选择**属性**
    2. 选择 详细信息 ，属性 设置成 设备实例路径，下面的值就是 设备实例 ID，可以右键复制
    3. 在 CMD 命令行窗口，输入一下命令就可以卸载对应的设备，关于禁用相关的命令可以参考 官方文档
    1

      pnputil /removedevice "设备实例ID"
    1. 可以看到已经卸载

    # 辅助

    1. 使用这种办法，是因为他可以禁用，也可以卸载
    2. 其次，命令行可以使用三方工具，能够更加便捷
    3. 我是使用的 quicker 工具，做成命令，直接按就可以
    4. 也可以写成 bat 批处理文件，每次开机运行，右键使用管理员运行即可****
    5. 当然，如果是禁用设备的，就不需要重复运行；卸载设备才需要每次开机运行，因为我的设备无法禁用，只有卸载

    # 其他方法

    在设备管理器更新驱动，然后选择一个错误的驱动文件， 可以参考以下文章 https://www.51mitang.com/zhishi/163112.html
