自建七牛云图床

最近编辑于 Unknown

    1. 使用图床的原因是方便迁移，但是缺点是，需要网络；
    2. 对于自己的知识库来说，图床是完全没有必要的，因为本地可以随时打开，比网络加载要好很多；
    3. 但是对于需要分享的内容来说，图床则是有很多的优点，目前市面上的笔记几乎都支持 markdown。如此，图床的图片链接就方便的很多。说白了，就是方便迁移
    4. hexo 以及一些自建的服务，图片链接的路径可能采用了不同的规则。比如 hexo 需要使用项目根路径作为相对路径，更多的则是采用当前文件为相对路径的起始路径。但是网络 url 几乎是所有都兼容的。这大致就是图床的必要性吧

    # 所需

    1. 七牛云
    2. picgo，手机端可以采用小白图床
    3. 域名，方便以后图片迁移

    使用七牛云的原因是，域名不用备案，同样有https协议。可以采用三方cloudflare托管域名，可玩性更高，不过可能速度会可能会变慢。

    # 域名配置

    这一步不做，就没必要自建图床。

    自定义域名之后，以后服务到期或者你需要迁移的时候，直接将图片全部下载下来，然后换到别的对象存储提供商，域名绑过去，原来的图片链接还可以用。很方便。 我的资源桶是国外的，因为我的域名没有备案。

    1. 进入你的资源桶
    2. 点击域名管理，添加自定义域名，不是 cdn 域名，这个是加速的，如果需要可以后面设置（不设置也完全够用）。
    3. 绑定域名
    4. 会出现 cname 绑定，然后在你的域名服务提供商中添加 cname 记录即可
    5. 点击空间管理，选择你刚才添加的域名
    6. 这样你的图片域名就是你自己添加的了
    7. 当然，你还可以给自己弄个 ssl 证书，七牛云有免费的单域名证书。

    关于迁移方案 目前我觉得不错的有

    1. 腾讯云 cos ： 腾讯云配置教程 腾讯云有免费半年的 cos
    2. 阿里云 oos 阿里云配置教程

    # picgo 相关

    此工具用于自动上传，常常与一些写作或者笔记工具的插件结合使用，就不需要手动切换了 相关的一些软件，相关的配置教程网上以及相关论坛都能找到

    1. typora markdown 可以说是最好的编辑器
    2. obsidian 知识库软件（当然也可以说是笔记软件）
    3. quicker 相当于 win 端的 ios 快捷指令，用于一些命令和小工具的快捷使用，非常方便。可以结合使用，上传图片
