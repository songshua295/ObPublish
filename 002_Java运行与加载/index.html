# 002_Java运行与加载

[TOC]

# 两个阶段：

  1. 编译 javac *.java

    1. java将源代码执行，生成JVM可以识别的字节码
    2. 源代码不能直接执行
    3. 一次编译到处运行

  2. 运行 java 类名(不要加文件后缀名)

源代码文件：.java 字节码文件：.class(不是二进制，因为如果是二进制，就可以直接被操作系统识别，就不需要JVM了)

# 大致过程：

使用Javac编译源代码.java文件，生成.class字节码文件，通过JVM将字节码进行解释为二进制文件，然后JVM将生成的二进制文件交给操作系统，操作系统与硬件进行交互。

当java HelloWorld之后发生了生么

  1. 启动类加载器classloader，类加载器在硬盘上查找HelloWorld.class文件。找不到就会报错“找不到或无法加载主类”
  2. 类加载器找到对应的类文件，将类的字节码文件装载到JVM中
  3. JVM对字节码进行解释，然后将解释之后的二进制编码传给操作系统，操作系统可以直接识别执行

# 注意事项

注意：.class也是一种特殊的二进制文件，只是不能直接被机器识别和执行，需要通过JVM进行解释后传给操作系统才能执行。

java命令使用时，一般是在当前路径下去查找对应的类文件，但是也可以改变查找的路径，通过别的路径去进行查找类文件。 查找类文件到别的文件查找，可以修改003.环境变量classpath，修改之后只会到那个目录进行查找，可以，但没必要。 在有些版本的JDK中，不需要javac，可以直接使用java 源代码文件名.java就可以直接运行代码，并且不会生成字节码class文件，这是新的java jdk的新特性，为了简化开发流程。
